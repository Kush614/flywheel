🔁 Flywheel

Ads that rewrite themselves
to fit your users.

An agent loop watches how users respond to each ad, keeps the winners, and regenerates the ads users ignore — the concept, the image, and the video — then ships the winner to every ad platform. Two sponsors do the heavy lifting.

The loop, end to end

👥
Real users
click and convert on your ads. That behaviour is the ground truth the loop optimizes — nothing the agent can fake.

Nexla — user response, governed observe
The response arrives as a governed Nexset — schema-enforced, validated, lineage-tracked — exposed to the agent over an MCP toolset. The loop's observe step trusts only this.
▼
🔁
Flywheel — the loop engine plan · act · observe · correct
Reads the response, diagnoses each ad, and self-corrects. It knows the difference between "bid too low" and "the creative doesn't resonate," picks a different fix per cause, and halts itself instead of thrashing on noise.
▼an ad flops (users click, don't convert)
🎨
Adbox — decide → render act
Decide: draft 3 fresh concepts from the ad's context, pick the strongest. Render: generate a new image, then animate it into a hero video. This is Adbox's creative factory — re-architected off Akamai.
▼powered by
⚡
Zero — keyless compute, paid per use x402 wallet
The agent discovers and pays for the LLM (concepts), the image model, and the video model through Zero — no API keys, no Akamai, pennies from an x402 wallet. Every spend is gated by Pomerium.
▼an ad wins for real users
📤
Nexla — publish the winner act → ship
The winning ad becomes a governed campaign and Nexla ships it to Shopify, Google Merchant, and Meta Catalog — one source, many validated destinations. That's Nexla's core job, both directions.
🛡 Pomerium wraps every money-spend. A per-session budget cap, human-approval-above-threshold, and an infra-layer gate that blocks prompt-injection (a poisoned "bid $10,000" field is refused below the agent's reasoning — it physically cannot overspend). Every decision is audited.

How each sponsor is used — their core capability

◐ Nexla — the data plane

Governed data in (user-response Nexset the loop reads) and out (winning campaign shipped to ad platforms via a governed pipeline). Schema, validation, lineage, MCP toolset — data the agent can trust.

⚡ Zero — keyless compute

Discover + pay (x402) for any capability with no pre-wired keys. Here it brokers the whole compute stack — LLM concepts, image rendering, and video — replacing Akamai. It even hosted this page's sibling site for free.

🎨 Adbox — the creative engine

The "decide → render" factory (draft concepts → pick → render). Adbox alone renders and stops; Flywheel adds the measurement — did the new ad actually lift user response? — so creative becomes a measured optimization.

🔁 Flywheel — the loop

A generic loopkit engine: plan → act → observe → correct, with a thrash/stop detector and an explore/exploit budgeter. The same engine also runs a second domain unchanged — proof it's real loop engineering.
